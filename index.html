About RTL Forge

RTL Forge is a web-based AI-assisted RTL development platform for early-stage semiconductor design. It combines a deterministic rule-based validation engine with an AI inference layer to detect structural Verilog errors, generate synthesizable code from natural language descriptions, and provide design guidance.

The platform supports Detect & Fix (paste code → run 8 rule checks → get auto-corrected output) and Generate RTL (describe a module → get synthesizable Verilog in a new page with full analysis). Final validation via linting, simulation, and synthesis tools remains the engineer's responsibility.

Rule Engine — All 8 Active Checks
R01 · BLOCKING_IN_SEQ
Blocking in Sequential Block
Detects = instead of <= inside always @(posedge clk). Causes simulation-synthesis mismatches.
ERROR
R02 · MISSING_RESET
Missing Reset Logic
Checks that every sequential always has reset handling. Flip-flops without reset have undefined initial state.
WARNING
R03 · INCOMPLETE_CASE
Incomplete Case Statement
Flags case statements missing a default clause — triggers latch inference in combinational logic.
ERROR
R04 · LATCH_INFERENCE
Latch Inference Risk
Detects if without else in combinational always blocks — missing coverage creates unintended latches.
WARNING
R05 · SENS_LIST
Explicit Sensitivity List
Warns when always uses an explicit signal list instead of @(*). Can cause simulation mismatches.
WARNING
R06 · FSM_STATE_REG
FSM State Register Error
Detects FSM state register using blocking assignment — causes race conditions in state transitions.
ERROR
R07 · COUNTER_OVERFLOW
Counter Overflow Check
Warns when counter increment found without visible wrap/max-count condition.
WARNING
R08 · MULTI_DRIVER
Multiple Driver Detection
Identifies signals assigned in too many locations — potential multiple drivers causing X-propagation.
WARNING
Supported RTL Categories

Counters
Up/Down/Bidirectional/Modulo/Gray/Johnson. Sync & async reset, enable, load.
◉
FSMs
Moore/Mealy. Traffic light, vending machine, sequence detector. 2 & 3-always styles.

SISO/PISO/PIPO. Johnson ring. Bidirectional with direction control.
⊕
ALU Blocks
4–32 bit. Configurable: ADD, SUB, AND, OR, XOR, NOT, SHL, SHR.
⊞
Combinational
Mux, decoder, encoder, priority logic, custom blocks.
⊗
Restricted
AXI/CPU pipelines/cache/SoC — not in v3 prototype.
